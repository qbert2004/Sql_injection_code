# Оценка качества системы

## 1. Постановка задачи

**Цель:** Построить систему обнаружения SQL-инъекций с точностью ≥ 99% и FPR < 2%,
пригодную для защиты web-приложений в продуктивной среде.

**Классы:**
- `0` (SAFE) — легитимный запрос
- `1` (INJECTION) — SQL-инъекция

**Приоритеты:**
- FPR (false positive rate) критичен — ложная блокировка легитимного пользователя недопустима
- FNR (false negative rate) важен — пропущенная атака опасна
- Баланс достигается через ансамблирование: высокий recall CNN + высокая precision RF

## 4. Ансамбль: обоснование весов

```
P_ensemble = 0.65 × P_cnn + 0.35 × P_rf
```

| Вес | Аргумент |
|-----|---------|
| CNN = **0.65** | FNR CNN (0.147%) на порядок лучше RF (1.68%) — обнаружение атак важнее |
| RF = **0.35** | FPR RF (0.023%) лучше CNN (0.055%) — RF удерживает precision ансамбля |

**Итог ансамбля (тест):** F1 ≈ **99.3%**, FPR ≈ **1.5%**

Ансамбль занимает позицию между двумя моделями: лучший recall чем у RF,
лучший FPR чем у чистого CNN — оптимальный компромисс для WAF-сценария.

## 5. Ablation study — вклад слоёв

| Конфигурация | Ложных тревог | Пропущенных атак | Статус |
|-------------|:---:|:---:|:------:|
| Только лексика (слой 1) | низкие | **высокие** | ❌ |
| Только RF (слой 2) | **0.023%** | 1.68% | ⚠️ |
| Только CNN (слой 2) | 0.055% | **0.147%** | ⚠️ |
| RF + CNN ансамбль | ~0.04% | ~0.5% | ✅ |
| Ансамбль + семантика (слой 3) | **< 0.02%** | ~0.5% | ✅✅ |
| Полный 7-слойный пайплайн | **< 0.02%** | **< 0.5%** | ✅✅✅ |

**Ключевой вывод:** семантическая валидация (слой 3) является критической — она устраняет
ложные срабатывания ML на безвредных строках с SQL-лексемой (например, поле комментария
с текстом "select the option").

## 8. Ограничения

1. **Zero-day обфускация** — новые методы обхода (не в обучающей выборке) могут снизить recall
2. **Специфика диалекта** — sqlglot хорошо работает с MySQL/PostgreSQL/SQLite; экзотические диалекты (Teradata, Snowflake) могут снизить semantic_score
3. **Многоязычные строки** — Unicode-атаки за пределами NFKC нормализации не тестировались
4. **Длина входа** — VDCNN ограничен 200 символами; длинные запросы усекаются (риск пропуска атаки в хвосте)
5. **Онлайн-обучение агента** — SGDClassifier накапливает дрейф при высоком потоке ложных позитивов
